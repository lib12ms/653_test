### 4-4. `should_keep_keyword` 과다차단 수정 (`preprocess.py`)

**수정:** 키워드가 금지어에 *포함*되는 방향 체크 제거. 금지어가 키워드의 *접두어*인 경우만 차단.

# 수정 후: 정확일치 + 금지어가 키워드의 접두어인 경우만 차단
if compact == tok_compact:                              return False
if len(tok_compact)>=3 and compact.startswith(tok_compact): return False

## 6. ISBN 부가기호 KDC 라우팅 — 구현 완료

### 6-1. 스파이크 결과 (2026-07-03)

| 확인 항목 | 결과 |
|---|---|
| NLK search.do API | 부가기호(EA_ADD_CODE) 제공 **안 함** |
| Seoji API (`SearchApi.do`) | `EA_ADD_CODE` 필드에 5자리 부가기호 **포함** |
| Seoji `KDC` 필드 | 실응답 대부분 **비어 있음** |

→ **EA_ADD_CODE 마지막 3자리**가 내용분류코드로 사용. KDC 필드는 보조.

---

### 6-2. 파이프라인 변경 전/후

**이전:**
```
Aladin 카테고리 → get_category_group() → 프롬프트 선택
```

**이후:**
```
Aladin 카테고리 ──────────────────────┐
                                      ↓
Seoji EA_ADD_CODE → content_code → get_category_group() → 프롬프트 선택
                                 (Aladin이 "기타"·"인문학"이면 KDC 우선)
```

---

### 6-3. 구현 내용 (5개 파일)

**`models.py`**
```python
class AladinMetadata653(BaseModel):
    ...
    content_code: str = ""   # 신규: 부가기호 마지막 3자리(내용분류코드)

class NlkMetadataHint(BaseModel):
    ...
    ea_add_code: str = ""    # 신규: Seoji EA_ADD_CODE 5자리 원본
```

**`nlk_metadata.py`**
- `hint_from_seoji_doc()` 수정: `EA_ADD_CODE` 파싱 추가
- `content_code_from_hint()` 함수 신규: EA_ADD_CODE 뒤 3자리 우선, 없으면 KDC 앞 3자리 대체

**`nlk_client.py`**
- `fetch_kdc_content_code_by_isbn()` 신규: Seoji를 직접 호출해 content_code만 반환하는 경량 함수. `nlk_enable`/`nlk_api_key` 필요.

**`metadata_merge.py`**
- `merge_aladin_with_nlk(... content_code="")` 파라미터 추가
- 반환 `AladinMetadata653`에 `content_code` 포함

**`ai_service.py`**
- `_KDC_CONTENT_CODE_RANGES` 구현 (KDC 강목 → category_group 매핑 테이블)
- `kdc_content_code_to_group()` 함수 신규
- `_category_group_from_text()` 분리 추출 (기존 Aladin 텍스트 로직)
- `get_category_group(category, content_code="")` 파라미터 추가
- `content_code` 파라미터가 `generate_653_subfield_line`, `finalize_653`, `_build_input`, `_extract_category_candidates` 전체 파이프라인에 전파

**`batch_653.py`**
- `fetch_kdc_content_code_by_isbn` 호출 추가
- CSV 출력에 `부가기호`, `분야그룹` 컬럼 추가

**`main.py`**
- `asyncio.gather()`에 `content_code_task` 추가 (Aladin·KPIPA와 병렬 호출)

> "시 분야를 별도 분야로 만들어 작업하는 것이 유의미할까요?"
