01전체 요청 흐름

/api/field653 한 번의 호출 안에서 벌어지는 일입니다. 파란 계열은 외부 API 병렬 조회, 초록 계열은 알라딘 중심의 주 경로, 주황 계열은 후처리·검증 단계입니다.

주 경로 — 알라딘 메타 · 분류 · 프롬프트
보조 출처 — KPIPA · Seoji(선택적, 플래그로 on/off)
후처리 · 품질 검증
진입점

Streamlit UI / batch_653.py / 외부 클라이언트

단건 조회, 96권 단위 배치, 또는 다른 서비스가 직접 POST /api/field653 호출

1 · 메타데이터 병렬 수집

알라딘 (필수)

ItemLookUp API로 분류·제목·저자·설명·목차. 출판사 책소개는 API에 없어 상세페이지를 직접 크롤링해 보완.

aladin_client.fetch_aladin_for_653()

KPIPA (선택)

ONIX 목차(TextType 04)만 사용. kpipa_enable 꺼져 있으면 즉시 빈 값.

kpipa_client.fetch_secondary_metadata_hint()

Seoji 부가기호 (선택)

ISBN 부가기호(EA_ADD_CODE) 마지막 3자리 = 내용분류코드. nlk_enable 꺼져 있으면 즉시 빈 값.

nlk_client.fetch_kdc_content_code_by_isbn()

세 호출은 asyncio.gather로 동시에 실행 — 순차 호출 대비 지연시간 절감

2 · 병합 + 전처리

metadata_merge.merge_aladin_with_nlk()

알라딘을 기준으로 KPIPA 목차 보강, 부가기호 content_code 부착. 동시에 preprocess.py가 카테고리 유통어 제거, 설명에서 저자 이력 문장 제거, 목차를 장 단위로 압축.

3 · 분야 라우팅

ai_service.get_category_group()

알라딘 카테고리 텍스트로 18개 그룹 중 하나 결정 → "기타"·"인문학" 캐치올일 때만 부가기호로 보정. (세부 로직은 별도 분야 라우팅 다이어그램 참고)

4 · AI 생성

_build_input() → OpenAI Responses API

  1. 고정 _STATIC_INSTRUCTIONS — 4대 원칙(독립성·구체성·목적성·시의성), 모든 분야 공통
  2. 분야별 동적 지침 — CATEGORY_PROMPTS[group]의 유형A-E 슬롯
  3. 참고 예시 — few_shots.json에서 같은 그룹 사서 작성 사례 최대 3개
  4. 도서 정보 — 분류·제목·저자·설명·목차·출판사 책소개
5 · 후처리 · 품질 검증

finalize_653() — 3단 계단식 보강

  1. 필터링 — 제목·저자 단어 금지, 저효용어·상위분류명·감상어 등 다수 규칙으로 차단
  2. 텍스트 fallback — 유효 키워드 0개일 때만, 설명·목차에서 명사 추출(문학 그룹은 제외)
  3. 카테고리 fallback — 그래도 min_keywords 미달이면 분류 체인의 구체 하위분야명 사용

결과로 Field653Quality(품질점수·경고플래그) 산출 — fallback을 썼다는 사실 자체가 감점 요인

03모든 분야가 공유하는 품질 4대 원칙

분야별 유형A-E 프롬프트 위에, _STATIC_INSTRUCTIONS로 모든 호출에 고정 전송되는 상위 규칙입니다.

01

독립성

제목·저자에 쓰인 단어를 그대로 반복하지 않는다. 하위개념으로 치환은 허용.

02

구체성

과학·인문학 같은 상위분류명 단독 출력 금지. 반드시 구체 하위개념으로.

자연과학 → 양자역학
03

목적성

이용자가 검색창에 실제로 입력할 명사만. 감상어 금지, 판촉어는 검색 효용이 있으면 허용.

위로 → 번아웃
04

시의성

신조어·트렌드어는 적극 허용한다.

LLM · N잡러 · 챗GPT
