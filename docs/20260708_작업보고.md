# 653 자유주제어 생성 시스템 — 작업 보고
**작업일: 2026-07-08**

---

## 1. 작업 배경

배치 결과 CSV를 분야별로 나눠 실제 출력 키워드를 하나하나 검토하면서 문제 패턴을 찾고, 발견 즉시 코드에 반영 → 재검증하는 방식으로 진행했습니다. 오늘 검토한 분야: 생활실용, IT컴퓨터, 경제경영, 교육, 문학, 사회정치.

---

## 2. 핵심 발견 — "프롬프트 예시 단어 그대로 복붙" 패턴

오늘 발견한 문제 중 가장 광범위하고 반복적으로 나타난 것은 **AI가 카테고리별 프롬프트에 적어둔 예시 단어를 실제 내용과 무관하게 그대로 베끼는 현상**이었습니다. "설명에 명시된 경우만"이라는 조건이 붙어 있어도 예시 자체가 완성된 답 형태라 그냥 골라 쓰는 경향이 있었습니다.

### 실제 사례

| 분야 | 새어나온 예시 단어 | 실제로 무관했던 책 |
|---|---|---|
| 생활실용(요리) | `한식조리` | 프랑스 요리책에 붙음 |
| 경제경영 | `디지털전환`·`중간관리자`·`직장인재테크` | 블로그 마케팅 에세이, 마케팅 서적 등 |
| 교육 | `대안교육` | PLC 공학교재, 교사 AI 도구 가이드, 중국학 논문집 등 5권 |
| 문학 | `정체성혼란` | 조선왕후 역사소설·연암 박지원 풍자소설·일본소설 등 8권 |

### 조치

1. **모든 분야(17개) 프롬프트 공통 서두에 지침 한 줄 추가** (일괄 반영):
   > "예시 단어는 형식 참고용일 뿐 정답이 아님 — 책 내용과 무관하면 예시를 그대로 쓰지 말 것."
2. **문학 프롬프트에서 "정체성혼란"을 "권력투쟁"으로 교체** — 재검증 결과 3연속 재발이 없어져 남용이 가장 심했던 사례를 근본적으로 없앴습니다.
3. 경제경영/교육 쪽 예시 단어(디지털전환 등)는 지침 추가만으로 재검증 시 뚜렷이 개선되어 별도 교체 없이 유지했습니다.

**검증 결과**: 재검증한 사례 대부분(경제경영 4/4, 문학 4/4, 교육 4/5)에서 오매칭이 사라지고 훨씬 구체적인 키워드로 대체됐습니다.

## 3. 프롬프트 자체 개선 — 생활실용 / IT컴퓨터

배치 데이터를 보다가 발견한, 프롬프트에 아예 대응 슬롯이 없어서 생긴 구멍들입니다.

| 발견 | 조치 |
|---|---|
| "원예"·"반려동물" 카테고리 책에 상위분류명이 그대로 나오거나 동의어만 반복 | 생활실용 유형A-C에 `원예이면)`·`반려동물이면)` 하위유형 신규 추가 |
| "건축" 단독 키워드 노출 (건축이면 하위유형은 있는데 금지어엔 빠짐) | 유형A 금지어에 "건축" 추가 |
| 프랑스 요리책에 "한식조리" 오매칭 | `나라별요리이면)`·`주류/음료이면)` 하위유형 신규 추가 |
| 뷰티·헤어 카테고리 대응 슬롯 없음 | `뷰티이면)` 하위유형 신규 추가 |
| "인공지능" 관련 도서 대부분이 첫 키워드로 "인공지능"만 반복 | 처음엔 단독 금지를 시도했다가, **"중요한 키워드를 아예 막는 건 과함"**이라는 판단으로 철회 → **"인공지능/AI가 들어간 키워드는 최대 2개까지"로 코드에서 개수 제한** (프롬프트 지시가 아니라 `finalize_653()`에서 직접 카운트해 강제) |

## 4. 후처리 필터 확장

카테고리 체인이나 문서 구조에서 새어나오는 무의미한 키워드를 계속 발견해서, **개별 단어 등록 대신 일반 규칙**으로 정리했습니다.

| 필터 규칙 | 잡아내는 사례 |
|---|---|
| `~계열` 접미 전체 차단 | 의약학간호계열·농축산생명계열 (경상계열 등 개별 등록 방식에서 일반화) |
| `~일반` 접미 전체 차단 | 북한학일반 (경영일반·음악일반 등 기존 개별 등록도 이 규칙 하나로 흡수) |
| "서문"·"기타"·"공예" 등 단독어 추가 | 문서구조어·카테고리 말단이 그대로 노출되는 사례 |
| 텍스트 fallback 조사 필터 확장 | 기존엔 "~는"·"~면"만 걸렀는데 "~의·이·가·을·를·은·들·과·와·며"로 끝나는 문장 조각("없는"·"길의"·"나침반이"·"사람들")도 차단하도록 확장 |

## 5. 데이터 수집 로직 개선

**알라딘 책소개(description)가 150자 미만이면 출판사 제공 책소개(publisher_desc)를 병합**해서 함께 취급하도록 `aladin_client.py`를 수정했습니다.

- 계기: 위스키 도감 책이 책소개는 2문장뿐인데 출판사 책소개엔 "위스키 르네상스" 같은 좋은 소재가 있었지만 프롬프트에서 활용이 안 되던 문제
- 효과: 해당 책 품질점수 0.136 → 0.714로 개선
- Streamlit UI에도 "책소개가 짧아 출판사 책소개를 병합했습니다" 배지 추가

## 6. AI 거절 응답 자동 재시도

콘텐츠에 따라 OpenAI가 이따금 거절 응답(`I'm sorry, I can't assist...`)을 내는 게 확인됐고(1920년대 출판검열사, 정원 에세이 등 특별히 민감하지 않은 책에서도 발생 — 비결정적 현상), 거절 시 곧장 텍스트 fallback으로 떨어지면서 품질이 크게 떨어지는 문제가 있었습니다.

**→ 거절 감지 시 1회 자동 재시도하도록 수정.** 재시도해도 거절이면 그때 fallback으로 전환. 토큰 사용량도 두 호출분을 정확히 합산해 기록합니다.

재시도 적용 전/후 비교(같은 3권 재검증):

| 책 | 재시도 전 | 재시도 후 |
|---|---|---|
| 그들은 그렇게 말하지 않았다 | `교양·이해·망망대해에서·인간·무한도전` (문장 조각) | `비판사회학·언론학·명언·역사왜곡·정보조작` |
| 가든 타임 | `교양·찾아·발견·정원·안에서` (문장 조각) | `정신건강증진·환경심리학·마음챙김·도시정원·생활환경` |
| 평양에 가보진 않았습니다만 | `통일·북한관계·북한학·코로나·이후` | `북한연구·관광사회학·국제관계학·북한사회변화·관광정책` |
