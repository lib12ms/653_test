## 1. 알라딘 API 연결 확인

**문제**: 로컬 네트워크의 SSL 인터셉션으로 인해 알라딘·NLK·OpenAI 전 API에서 `CERTIFICATE_VERIFY_FAILED` 오류 발생

**조치**:
- 진단/배치 스크립트에 `httpx.AsyncClient(verify=False)` 적용
- `Settings(allow_insecure_ssl_fallback=True)` 로컬 전용 설정
- 프로덕션 코드의 SSL 우회 로직(`_can_use_insecure_fallback`)은 `.env` 값으로만 활성화되도록 유지

## 2. 편차 패턴 진단 (`diagnose_653.py` 작성)

8개 장르별 ISBN을 대상으로 전체 파이프라인을 실행하고, 입력 품질과 출력 품질을 비교하는 진단 스크립트를 작성했다.

**측정 항목**:
- 목차 길이 / 설명 길이 / 입력 풍부도
- NLK 데이터 유무 (KDC 분류번호, 키워드, 목차·설명 보강 여부)
- AI 생성 키워드 수 vs. 후처리 후 최종 키워드 수
- backup 후보 사용 여부 / 주의 플래그

**진단 결과 (로컬 환경 기준)**:
- 목차 데이터: 전 도서 0자 (로컬 SSL 차단 환경 특성, 실 서비스에서는 대부분 제공됨)
- backup 사용: 0건 → 프롬프트 수준에서 충분히 커버됨
- 키워드 부족: 0건

**결과물**: `backend/diagnose_653.py`, 실행 시 타임스탬프 CSV 자동 저장

## 4. 배치 처리 스크립트 (`batch_653.py` 작성)

ISBN 목록을 일괄 처리하고 결과를 Excel 호환 CSV로 저장하는 스크립트를 작성했다.

**기능**:
- ISBN 텍스트 목록 → 알라딘·NLK 수집 → AI 653 생성 → CSV 저장

## 5. 국가+장르 키워드 필터

**문제**: AI가 `한국문학`, `한국소설`, `일본소설`, `한국에세이` 등 국가명+문학장르 복합어를 생성 — 이는 서지 분류어로 독자 검색 키워드로서 가치가 낮음

**해결**: 프롬프트 규칙 + 코드 필터 이중 적용

```python
# backend/app/ai_service.py
_COUNTRY_GENRE_RE = re.compile(
    r"^(한국|일본|영미|중국|프랑스|독일|러시아|미국|영국|스페인|이탈리아|북유럽)"
    r"(문학|소설|에세이|시|희곡|단편소설|장편소설|수필|동화|산문|시집|소설집|산문집|문예|시문학)$"
)
```

- 프롬프트 [2단계: 필터링]에 명시적 제외 규칙 추가
- `_is_low_value_keyword()`에 정규식 체크 추가 (코드 안전망)

**차단 예시**: 한국문학, 한국소설, 한국시, 일본소설, 영미소설  
**통과 예시**: 한국사, 한국사회, 한국불교, 성장소설, 추리소설

## 6. 키워드 부족 케이스 개선

**문제**: 국가+장르 필터 적용 후 일부 문학 도서에서 카테고리 backup 후보도 전부 차단되어 키워드가 1~3개로 급감

**원인 추적**:
- `겨울통` (ISBN 9791167376565): 설명이 수상·이력 문장으로만 구성 → `clean_description_for_ai`에서 대부분 제거 → AI에게 전달 정보 거의 없음 → `언어탐구 + 한국소설` 2개 생성 → `한국소설` 차단 → 1개만 남음
- 카테고리 체인: `소설/시/희곡 > 한국소설 > 2000년대 이후 한국소설` → backup도 `한국소설`뿐 → 차단

**해결**:

```python
# 장르별 대체어 fallback (카테고리 backup이 전부 차단될 때만 발동)
_GENRE_FALLBACKS: dict[str, list[str]] = {
    "소설": ["현대소설", "장편소설"],
    "시":   ["현대시", "서정시"],
    "에세이": ["에세이"],
    "희곡": ["현대희곡"],
    "동화": ["어린이문학"],
    "만화": ["그래픽노블"],
}
```

발동 조건 (두 가지 모두 충족 시에만):
1. AI 생성 유효 키워드 < 5개 (min_keywords)
2. 카테고리 체인의 모든 후보가 필터에 차단

**결과**:

| 도서 | 이전 | 이후 |
|---|---|---|
| 겨울통 | 1개 (`언어탐구`) | 5개 (`언어탐구 / 문학적조각 / 감정서사 / 현대한국소설 / 현대소설`) |
| 알면 재미있지 않나요? | 3개 | 5개 |

## 7. FEW_SHOT_NO_TOC 코드 제거

목차 없는 케이스 대응을 위해 구현했던 분야별 few-shot 예시 딕셔너리를 제거했다.

**제거 배경**: 실 서비스에서는 알라딘 API를 통해 대부분의 도서에 목차가 제공됨. 목차가 없는 경우는 소수 예외이며, 프롬프트 최적화 이후 기본 지침만으로도 충분히 커버됨이 확인됨.

**제거 내용**:
- `FEW_SHOT_NO_TOC` 딕셔너리 (10개 분야 × 예시 1건, 54줄)
- `toc.strip() == ""` 조건부 주입 로직 (5줄)
